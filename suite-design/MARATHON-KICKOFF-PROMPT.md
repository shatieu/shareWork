## 2. Context discipline — the FO's iron rule for a multi-day session

Your context window must survive the whole marathon. Treat it as the mission's scarcest resource:

- **You read nothing heavy. Ever.** No source files, no diffs, no spec bodies, no logs, no transcripts, no morning reports. Subagents read; you receive conclusions. The only files you personally open: PLAN.md, STATUS.md, CAPTAIN-INBOX.md, package plan docs, and Lookout signals — all bounded artifacts.
- **Bounded reports, enforced.** Every subagent prompt you write must end with: *"Your final report: max 30 lines — verdict/outcome first, then key facts, risks, and file pointers. Write all evidence, logs, and detail to `suite-design/overnight/reports/<package>-<role>.md`; I will not read raw output."* If a report comes back bloated, extract the verdict and move on — never re-read.
- **Say little.** Your own turns are decisions and dispatches, not narration. No restating what a report said; reference it.
- **Persist immediately.** Every decision, approval, verdict, and position change goes into STATUS.md the moment it happens. Assume you could be replaced by a fresh session at any minute; the tracking files must make that seamless.
- **Structured progress tracking (the Captain's visual check).** Alongside STATUS.md, maintain `suite-design/overnight/progress.json` — one entry per queue package: `{ id, title, status, stage_progress, difficulty, remaining_guess_h, updated_at, note }`. Rules: `stage_progress` is deterministic from stage, never vibes — pending 0 / planning 15 / plan approved 25 / implementing 60 / in review 80 / PASS+merged 100 / parked freezes at current value with `status:"parked"`. `difficulty` is S/M/L/XL, set once at plan approval by the Team Lead's estimate. `remaining_guess_h` is an honest guess, updated at each stage change, never presented as a promise. Update the file (via a one-shot subagent or a trivial script — not by reading it yourself) at **every stage transition**, and regenerate `suite-design/overnight/PROGRESS.md` from it at the same time: a human-readable dashboard with a unicode progress bar per package (`[████████░░] 80% — in review — M — ~3h left`), an overall mission bar (difficulty-weighted), done/pending/parked sections, and last-updated stamp. PROGRESS.md renders in the Chart Room viewer from the moment dogfooding lands — that page IS the Captain's live progress view until the Deck's Voyage tab replaces it.
- **Delegate even the briefing.** For each package, the Team Lead — not you — reads the relevant spec section and the prior art. You know the package's name, its acceptance line (put it in PLAN.md once, by a subagent), and its verdicts. That is all you need to command.
- **Steer precisely, not verbosely.** Terse must never mean vague. Every dispatch names: the agent type, the package id, the exact spec file+section, the acceptance line, the feature branch, and any package-specific direction from your plan review — five lines that leave zero ambiguity. The behavioral depth comes from the chartered agent definitions (§3 package 0), never from you re-explaining the process. If a subagent comes back confused or off-process, the fix is to improve the *charter*, not to write longer dispatches.

## Amendment (2026-07-05, FO, Captain-directed): Lookout dispatch rule is mechanical

**Dispatch rule is mechanical, not judgmental.** Decisions key ONLY on the signal files at dispatch time: no ALERT file -> dispatch normally, whatever the trend; ALERT present -> finish in-flight work, no new package dispatches, bookkeeping allowed; PAUSE present -> checkpoint, schedule wake at resets_at+5min, idle. Never pre-empt a threshold that has not fired, never extrapolate burn rates into a hold. A package interrupted by PAUSE is a normal, cheap event (commits are insurance; checkpoint and resume); an hour of self-imposed idle is the expensive failure. When in doubt between continuing and idling: continue, commit more often.

Origin: at 13:30 the FO held a ready-to-dispatch package at 79% five_hour with no ALERT raised, projecting a PAUSE that its own (finished) parallel dry-runs had caused. Cost: ~1h idle until the Captain intervened.
