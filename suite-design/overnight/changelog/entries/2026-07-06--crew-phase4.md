# 2026-07-06 — Crew (Bridge phase 4, package 8)

The ship-crew plugin grows from hooks-only to the full Crew (Ship_Spec §7/§9.4):

- **Six roles** (`plugins/crew/agents/`): first-officer (orchestrator, `--agent first-officer`),
  navigator, shipwright, inspector, devils-advocate, quartermaster — productized from the
  marathon's field-tested wave-* charters (30-line report contracts, plan fidelity,
  never-review-own-work, shared-worktree rules baked in).
- **Scrutiny presets + SessionStart wiring**: `hooks/scrutiny.mjs` resolves
  `ship.scrutiny` (settings.local wins; custom `ship.crewPresets`) and injects a `[Ship crew]`
  briefing — one settings line, zero further setup. Live-proven with a real `claude -p` haiku
  session naming preset, pipeline, gates, and all six roles.
- **Plan gate (rigorous) + paranoid stop gate**: `hooks/stop-gate.mjs` blocks the session's
  finish (`decision: block`) until the inspector's PASS marker
  (`.ship-crew/inspector-pass.json`) matches the session; loop-safe, fail-open.
- **Crew orchestration skill** (`skills/crew/SKILL.md`): pipeline per preset, dispatch format,
  hot-load fallback, stop-gate protocol.
- **Quartermaster MCP access**: new read-only `ship-log mcp` stdio server
  (`log_entries` with since/until ranges, `log_rollup`, `log_sessions`) beside the existing
  ship-ledger MCP; cross-week "where are we with X" tool chain proven end-to-end by
  `packages/ship-log/acceptance/quartermaster-mcp.mjs` (10/10 assertions, real stdio servers).
- Fix: seam READMEs moved out of `agents/`/`skills/` — every `agents/*.md` loads as a
  dispatchable agent (a bogus "README" agent showed up in live acceptance).

Tests: ship-log 81→88, new ship-crew-plugin suite 23; all floors hold.
