# Chart Room Phase 2 — Viewer (read-only)

Delivered a read-only browser viewer for Chart Room, per `ChartRoom_Spec.md` §8 build-order item 2:

- **Daemon** (`packages/chartroom/src/daemon/`): Fastify server, one `chokidar` watcher per
  registered repo (debounced reindex on change), a flat `~/.chartroom/repos.json` registry, and a
  JSON API (`/api/repos`, `/api/repos/:id/docs`, `/api/repos/:id/docs/:docId`) plus per-repo raw
  static asset mounts for images/doc source.
- **`chartroom serve` / `chartroom register`**: two new CLI subcommands. `npx chartroom serve`
  boots the daemon with zero required flags; `chartroom register <path>` adds a repo to the
  registry, idempotently.
- **New `packages/chartroom-ui` package** (private, React 19 + Vite 8): pretty markdown rendering,
  TOC, hand-rolled `<details>`-based section collapsing, backlinks, path/URL image handling,
  `:::llm`/`:::human` directive rendering (llm: always-visible tldr + collapsed body; human: plain;
  unrecognized directives degrade to visible plain content), and missing-link tombstone display.
  Its build output is copied into `chartroom`'s own publishable `dist/public`, so `npx chartroom
  serve` stays self-contained with no sibling-package dependency after publish.
- Backlinks and tombstone display are both direct reuses of phase 1's `indexer.ts`/`check.ts` —
  zero new detection logic, zero changes to the phase-1 index schema.

Verified: `tsc --noEmit`, `eslint .`, and `vitest run` clean in both packages (11 files/68 tests in
`chartroom`, 3 files/15 tests in `chartroom-ui`), both phase-1 and phase-2 acceptance scripts pass,
and a real `chartroom serve` process was booted against two throwaway registered repos and driven
with live `curl` (repo list, tombstoned-link data, raw asset fetch, path-traversal rejection —
`@fastify/static` returned 403 on both a plain and a double-encoded traversal attempt). Independent
adversarial review confirmed all of this plus scope discipline (no phase-1 file touched except
`cli.ts`'s two-line subcommand registration) and the read-only guarantee (only the registry file and
`.docs/index.json` are ever written; no doc file is ever written).

One residual gap, honestly flagged rather than hidden: the Chrome browser extension was not
connected in this session, so the planned real-browser click-through QA (repo switching, TOC
navigation, `:::llm` block collapse/expand) could not be performed. The daemon/API/build-bundle
proof above is real and live, but does not substitute for actually exercising the rendered React app
in a browser — logged in `DECISIONS-NEEDED.md` for a follow-up pass.
