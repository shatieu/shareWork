# Package 2 — Chart Room v1.1

Five deliverables on `ship-wave1-cr-v11` (17 commits; ~14 files salvaged from
the quarantined WIP branch `wip-quarantine-2026-07-05`, re-authored line-by-line
under the full new-code test bar):

- **Staleness rules in `chartroom check`** — opt-in frontmatter `ttl_days` and
  `sources:` freshness gates plus orphan detection (identified docs with zero
  inbound id-links). New `src/staleness.ts` engine (injectable clock/git seams),
  additive `staleness`/`stalenessClean` on `CheckResult` (`clean` keeps its
  exact link-integrity meaning; pre-commit hook byte-unaffected), CLI sections,
  `--json` block, exit 1 on ttl/sources violations, orphans warn-only unless
  `--fail-orphans`. Zero git subprocesses when no doc opts in.
- **Round-trip whitespace-gap fix** — the phase-3 quirk where a reorder plus
  non-default blank-line spacing leaked an original gap onto a new block
  adjacency; the positional gap-alignment fill is now reorder-aware
  (`roundTrip.ts`), with a repro test that failed pre-fix.
- **Real-agent e2e proof** (`acceptance/real-agent-e2e.mjs`, manual-run only) —
  a live headless `claude -p` in a scratch repo self-recovered a moved doc via
  the installed PostToolUseFailure hook (never told the new path) and an
  `:::ask-me` question round-tripped agent -> doc -> human answer -> resumed
  agent. Closes ChartRoom_Spec §9's last unproven DoD line.
- **`chartroom open <file>`** (+ `--print-url`) — resolves the owning repo
  (auto-registering never-seen git repos), finds a running daemon via new
  `~/.chartroom/daemon.json` (health-checked, stale-tolerant) or spawns one,
  live-registers new repos against a running daemon via new
  `POST /api/repos/register` (raw assets now served by a dynamic route instead
  of boot-fixed static mounts, so no restart), opens the browser at the doc's
  key-addressed URL.
- **`chartroom associate [--remove]`** (win32) — per-user HKCU ProgID +
  `.md\OpenWithProgIds` offer-only registration (never touches the `.md`
  default value or UserChoice), hidden `wscript.exe` VBS launcher with
  install-time presence check and PowerShell `-WindowStyle Hidden -File`
  fallback, SHChangeNotify after install/remove, all registry writes via
  no-shell `reg.exe` arg vectors (round-trip proven against a scratch HKCU key,
  unicode included). Demonstrated for real on this machine: the exact
  registered command opened a never-registered repo's spaced-filename `.md`
  end-to-end.

Substrate: daemon doc routes now address docs by key (`id ?? path`) via new
`doc-lookup.ts::findDoc` — id-less docs (most of any real repo) are viewable,
editable, checkbox-toggleable, ask-me-answerable, and asset-pastable; the UI
sidebar lists them and `DocDetail` carries `id`/`key`. Suites: chartroom
248/248, chartroom-ui 144/144, all six acceptance scripts green (new:
`open-associate-e2e.mjs` in the default chain).
