# Chart Room Phase 1 — Indexer, Resolver, Link Repair, CLI, Pre-commit Hook

Delivered the full phase-1 foundation at `packages/chartroom/` per `ChartRoom_Spec.md` §8.1:

- `.docs/index.json` schema + full-repo indexer (`src/indexer.ts`) with correct
  tombstone/move/resurrection diff logic — a doc keeps its identity across
  `git mv` (no tombstone), a truly deleted id is tombstoned, and a
  resurrected id clears its tombstone automatically.
- 5-step resolution algorithm (`src/resolver.ts`): id -> path-as-written ->
  unique filename -> fuzzy title (Dice coefficient, flagged `guess: true`) ->
  tombstone/not-found.
- Stale-link repair (`src/fix-links.ts`): AST byte-offset splicing, never
  touches frontmatter, never re-stringifies the file, ignores link-like text
  inside fenced code blocks. Defaults to report-only; `--write` required to
  mutate (a deliberate First Officer correction to the original plan — see
  `DECISIONS-NEEDED.md`).
- Pre-commit hook (`src/hook.ts` + `src/install-hook.ts`): operates on the
  staged git index blob directly (`git show :path` / `git hash-object -w` /
  `git update-index --cacheinfo`), never the working tree of a
  partially-staged file, never creates a commit itself, exits 0 to let the
  commit proceed. This was the single riskiest piece of the plan; it has a
  dedicated partial-staging test plus four other hook tests.
- `chartroom` CLI (`src/cli.ts` + `src/commands/*`): `init`, `index`,
  `resolve`, `fix-links`, `check`, wired via commander with exit codes
  exactly matching the plan's §8 spec.
- Acceptance script (`acceptance/git-mv-resolution.mjs`): proves the spec's
  phase-1 acceptance line end to end inside a disposable scratch repo — a
  `git mv`'d doc resolves via both the CLI and a raw index Read, and a
  pre-commit-hook pass normalizes only staged files, leaving unstaged edits
  and untouched docs byte-identical, with zero commits created.

Verified clean: `tsc --noEmit`, `eslint .`, `vitest run` (8 files / 53 tests),
and the acceptance script, all run live (not just reported) before merge.
Independent adversarial review confirmed the hook's git-plumbing,
resolver/indexer tombstone logic, and link-repair AST-safety line by line.

One gap found and fixed in the same package: nothing in the repo excluded
`.docs/` from git, so `chartroom init`/`index` would have committed the
generated index — added to the root `.gitignore`.
