# Chart Room Phase 3 — Editor (Milkdown, byte-identical round-trip, image paste, Ctrl+K)

Delivered in-place WYSIWYG editing per `ChartRoom_Spec.md` §8 build-order item 3, the highest-risk
phase so far since the spec itself names phantom diffs as a known WYSIWYG failure mode:

- **Block-level diff-and-splice round-trip engine** (`packages/chartroom-ui/src/editor/{segmentBlocks,
  roundTrip,opaqueNode}.ts`): the doc body is segmented into top-level blocks with byte offsets; each
  is classified as editable "prose" (fed to Milkdown) or protected "opaque" (directive/HTML/anything
  unrecognized — a strict allowlist, so unknown constructs default to protected, never mangled). On
  save, unchanged blocks splice back their **original raw bytes**, never Milkdown's re-serialization;
  only genuinely edited blocks get fresh output. Frontmatter is stripped before editing starts and
  always spliced back verbatim, unconditionally.
- **Milkdown** (`@milkdown/kit`, not the higher-level `@milkdown/crepe`) mounted via `@milkdown/react`
  in `DocEditor.tsx`, an edit-mode alongside the existing read-only `DocView`.
- **Image paste**: clipboard paste uploads to a new `POST .../assets` daemon endpoint, saves as
  `assets/<doc-id>/<timestamp>.png`, and inserts the relative link — reusing phase 1's existing
  content-hash asset registration with zero new indexing logic.
- **Ctrl+K link picker**: hand-rolled fuzzy-search modal (`fuse.js`) over the daemon's doc list,
  inserting the id-carrying link format at the cursor.
- **Save path**: a new `PUT .../docs/:docId` daemon endpoint, explicit Save (button + Ctrl+S) rather
  than autosave — matches the project's "never surprise the git diff" ethos. This is the first code
  in the whole project that writes to a `*.md` file from the daemon/UI side.
- **`fix-links.ts` extended** (the one behavior change to existing phase-1 code, approved and narrow):
  a second image-href-repair mechanism was added beyond the plan's original content-hash approach,
  closing a real bootstrapping gap the hash-only design couldn't close on its own — proven end-to-end
  by a new acceptance script (`editor-round-trip.mjs`: real scratch repo, real `git mv`, real
  `fix-links --write`).

Verified: 43 round-trip assertions (20 fixtures × no-op + single-edit, covering every named
construct plus GFM tables/task-lists/directives/raw HTML/frontmatter) all pass, both as `===` string
equality (not normalized comparison) for the no-op case and as a real line-level diff for the
single-edit case. `tsc`, `eslint`, and all three `chartroom` acceptance scripts (phase 1, phase 2,
phase 3) pass in both packages. An independent adversarial Reviewer additionally hand-wrote 23 novel
markdown constructs beyond the committed suite and confirmed byte-identical round-trip held for all
of them.

**One known gap, tracked not fixed:** the Reviewer found that reordering two existing blocks (drag/
cut-paste, not a text edit) can reformat one of them if its canonical form diverges from its original
authoring style (dash bullets, tilde fences, setext headings, etc.) — a formatting regression, not
data loss, and not the primary "edit-save produces zero diff on untouched content" flow, which held.
Logged in `DECISIONS-NEEDED.md` as a follow-up rather than silently accepted or hidden.

Same honest gap as phase 2: no real-browser click-through (Ctrl+K, paste events, Milkdown mounting)
was performed — the Chrome extension was unavailable again this session. Real builds, real `tsc`,
and real Fastify `.inject()`-driven acceptance scripts are the structural proxy used instead.
