# Package 12 — Sea Chest, code-complete (Locker_Spec phases 1–3)

New `packages/sea-chest`: the hosted-identity locker built to the seam — the future Harbor mounts
it, agents never touch the live platform. Items (skill/agent/hook/settings_template/claude_md/
mcp_config/preset/plugin_bundle) are stored plugin-shaped per user with append-only version
history; `SeaChestStore` ships as a full-fidelity memory implementation plus a
`SupabaseSeaChestStore` mapping over a *structural* PostgREST client (zero supabase-js
dependency — Harbor passes its own client).

Phase 1: MCP tools `locker_list/pull/push/diff` (`registerSeaChestTools` for the platform's
existing `/api/mcp`; push bumps versions, identical content reports `unchanged`, diff returns
per-file unified diffs). Phase 2: private token-authed marketplace — docs research established
that URL-hosted marketplaces only HTTP-fetch `marketplace.json` while plugin files travel by
git/npm ONLY, so each item is also projected as a minimal npm package (deterministic dep-free
ustar+gzip tarball, packument, sha1/sha512 integrity) served from a per-user token-in-path
registry; manifest entries use the documented npm source type. Phase 3:
`locker_setup_machine` returns an executable setup manifest (marketplace add command, plugin
installs, write-if-absent file writes, service registrations) scoped by machine profiles, plus a
React locker page (browse by kind, version history, publish toggle, install snippet, profiles,
token mint/revoke) behind an injected client.

Supabase DDL ships as four migration FILES (RLS on everything: owner isolation, published-only
public reads, append-only versions, hash-only revocable tokens) with a throwaway-docker check
harness that loads them into real Postgres and proves the RLS behaviors — applied to nothing.
88 tests + a 10-check acceptance script (two-machine MCP round-trip, HTTP marketplace/registry
serving, setup-manifest execution into a temp home). Live seams (apply migrations, Harbor
mounts, real `/plugin install` against the custom registry) are parked in CAPTAIN-TODO /
DECISIONS-NEEDED.
